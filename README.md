# Manual QA Testing Project

## Overview
This repository contains a **manual QA testing project** designed to demonstrate core **software quality assurance skills** and best practices.

The project simulates a real QA workflow including test planning, test case design, bug reporting, and traceability, using GitHub as a documentation and collaboration tool.

---

## Project Objectives
- Apply manual testing methodology
- Create and structure a test plan
- Design clear and reusable functional test cases
- Document software defects with detailed bug reports
- Demonstrate QA traceability between test cases and bugs

---

## Test Scope
The current scope of testing focuses on the **Authentication module**, including:
- User login functionality
- Field validation
- Error handling
- Negative scenarios
- Basic UX and accessibility considerations

---

## Test Types
- Functional testing
- Validation testing
- Negative testing
- Exploratory testing
- Smoke testing
- Regression testin
